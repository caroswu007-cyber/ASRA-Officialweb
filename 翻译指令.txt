新架构：只有一条路径
docs/page-copy/*.xlsx（你编辑这里）
        │

        ▼
src/content/pageCopyDocs.generated.ts（自动生成，不要手动编辑）
        │
        ▼
getPageCopyText(locale, key, fallback)
        │   查找顺序：xlsx 中文/英文 → xlsx 英文 → TS 代码中的英文默认值
        ▼
  网页渲染中英文内容
你的日常操作
场景 1：修改现有文案（最常用）
打开 docs/page-copy/ 下对应的 xlsx 文件（如 01-首页.xlsx）
编辑 english_源文 列和/或 中文 列
保存 xlsx
运行 npm run copy:import
完成 — 网站上立刻反映修改
场景 2：网站新增了板块（偶尔）
运行 npm run copy:scaffold
在 docs/page-copy-staging/ 目录找到更新后的模板 xlsx
对比模板和正式 xlsx，把新增的行复制到正式 xlsx 中
填入中文翻译
运行 npm run copy:import
删除 staging 目录中的文件
两个命令的安全保证
命令	作用	安全性
npm run copy:import	读 xlsx → 写 generated.ts	绝对安全，永远不碰你的 xlsx
npm run copy:scaffold	读代码结构 → 写到 staging 目录	绝对安全，永远不碰 docs/page-copy/
旧的 export:page-xlsx（会覆盖你的 xlsx 的那个危险命令）已经被彻底删除了。
